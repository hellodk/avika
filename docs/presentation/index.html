Why Push Over Pull?

Traditional (Prometheus)

Scrape every 15–60s → firewall/NAT issues, data gaps, inbound ports required.

Avika Push Model

Agent ↔ Gateway: persistent gRPC stream. Outbound-only, WAL buffer, bidirectional (commands, updates), 1s latency.

Agent ──(gRPC stream)──> Gateway
      <──(commands, updates)──

Result: Works behind NAT/firewalls, no data loss, single long-lived connection.
